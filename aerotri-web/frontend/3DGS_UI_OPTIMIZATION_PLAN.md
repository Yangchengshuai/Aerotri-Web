# 3D GS 前端页面设计优化方案

## 一、产品经理角度分析

### 1.1 用户场景分析
- **主要用户**：AI/ML工程师、研究人员
- **核心任务**：
  1. 配置训练参数
  2. 选择GPU资源
  3. 启动训练
  4. 监控训练进度和指标
  5. 查看训练产物
  6. 调试问题（查看日志）

### 1.2 信息架构优化
```
3DGS 模块页面
├── 顶部操作区（固定）
│   ├── 状态显示（状态标签 + 进度条）
│   ├── 快速参数（迭代次数、分辨率、数据设备、SH阶数）
│   └── 操作按钮（开始/中止训练）
│
├── GPU资源区（独立卡片，显眼位置）
│   ├── GPU选择器（单选）
│   └── 实时状态（显存、利用率）
│
├── 参数配置区（可折叠）
│   ├── 基础参数（已显示在顶部）
│   └── 高级参数（折叠）
│
├── 监控可视化区（训练中显示）
│   ├── TensorBoard（大尺寸预览 + 全屏选项）
│   └── 实时指标卡片（可选）
│
├── 训练产物区（始终显示）
│   └── 文件列表表格
│
└── 日志调试区（可折叠）
    ├── 训练日志
    └── 错误信息（如有）
```

### 1.3 用户体验原则
1. **信息层次清晰**：重要信息（状态、GPU）优先展示
2. **操作流程顺畅**：配置 → 选择GPU → 启动 → 监控
3. **减少认知负担**：避免重复信息，统一GPU显示
4. **响应式反馈**：实时更新状态和进度

## 二、交互设计角度分析

### 2.1 当前问题
1. **GPU显示重复**：GPU状态栏和3DGS模块中都有GPU信息
2. **信息密度过高**：所有信息堆叠在一起
3. **TensorBoard显示空间不足**：预览区域太小
4. **操作按钮分散**：不够集中

### 2.2 优化方案

#### 2.2.1 GPU资源统一管理
- **位置**：独立卡片，放在参数配置区之前
- **功能**：
  - GPU选择（单选）
  - 实时状态显示（显存、利用率）
  - 自动刷新（训练中）
- **移除**：高级参数中的GPU显示

#### 2.2.2 布局优化
使用卡片分组 + 标签页组合：
```
┌─────────────────────────────────────┐
│  状态 + 快速参数 + 操作按钮          │
├─────────────────────────────────────┤
│  GPU资源选择（独立卡片）            │
├─────────────────────────────────────┤
│  [参数配置] [监控] [产物] [日志]   │ ← 标签页
│  └─ 当前标签页内容                  │
└─────────────────────────────────────┘
```

#### 2.2.3 视觉层次
- **一级信息**：状态、进度、GPU选择（顶部固定）
- **二级信息**：参数配置、监控可视化（标签页）
- **三级信息**：详细日志、调试信息（可折叠）

## 三、前端工程师角度分析

### 3.1 组件复用
- **GPUSelector**：已实现，可直接使用
- **统一GPU显示**：移除重复的GPU状态显示
- **卡片组件**：使用Element Plus的el-card统一风格

### 3.2 布局技术方案

#### 方案A：标签页布局（推荐）
```vue
<el-tabs v-model="activeTab">
  <el-tab-pane label="参数配置" name="params">
    <!-- 参数配置内容 -->
  </el-tab-pane>
  <el-tab-pane label="训练监控" name="monitor">
    <!-- TensorBoard + 实时指标 -->
  </el-tab-pane>
  <el-tab-pane label="训练产物" name="outputs">
    <!-- 文件列表 -->
  </el-tab-pane>
  <el-tab-pane label="训练日志" name="logs">
    <!-- 日志内容 -->
  </el-tab-pane>
</el-tabs>
```

#### 方案B：卡片堆叠布局（当前）
- 优点：所有信息可见
- 缺点：页面过长，需要滚动

### 3.3 性能优化
- **懒加载**：TensorBoard iframe按需加载
- **防抖节流**：GPU状态刷新（已有2s间隔）
- **虚拟滚动**：日志内容过长时使用

## 四、具体优化实施

### 4.1 GPU资源区优化
- 位置：独立卡片，在参数配置之前
- 样式：突出显示，带边框高亮
- 功能：选择 + 状态显示一体化

### 4.2 TensorBoard优化
- 预览高度：600px（当前500px）
- 全屏对话框：95%宽度，calc(95vh - 100px)高度
- 新窗口选项：独立标签页打开

### 4.3 布局重组
- 顶部：状态 + 快速参数 + 操作按钮（固定）
- GPU区：独立卡片（显眼）
- 内容区：标签页切换（参数/监控/产物/日志）

### 4.4 响应式设计
- 桌面端：标签页布局
- 平板端：可折叠卡片
- 移动端：垂直堆叠，简化显示
